# YeVoda (Є вода?) 💧

**YeVoda** — це розумна система моніторингу рівня води на базі мікроконтролера **ESP32-C3** та безконтактного датчика рідини **XKC-Y25-NPN**. Пристрій призначений для встановлення на баки чи резервуари з водою та миттєво сповіщає про зникнення та появу води через приватного Telegram-бота.

> [!TIP]
> 📖 **[Читати детальну інструкцію користувача (MANUAL.md)](MANUAL.md)**

---

## 🌟 Основні можливості

- **Безконтактний моніторинг**: Датчик XKC-Y25-NPN монтується із зовнішньої сторони бака (не контактує з водою), що робить систему гігієнічною та надійною.
- **Інтерактивний Telegram-бот**:
  - Перевірка поточного статусу води за запитом за допомогою зручної кнопки **«Є вода?»**.
  - Можливість підписатися/відписатися від автоматичних сповіщень прямо в чаті за допомогою динамічних кнопок **«Підписатись на сповіщення 🔔»** та **«Не сповіщати 🔕»**.
- **Миттєві сповіщення**: Бот надсилає особисті повідомлення про зміну статусу води в реальному часі всім активним підписникам.
- **Енергонезалежна база підписників**: Система зберігає базу даних останніх 20 підписників у вбудовану пам'ять ESP32 (`Preferences`). При перезавантаженні чи знеструмленні підписки не зникають.
- **Оптимізована робота з Flash-пам'яттю**: Ввімкнення/вимкнення сповіщень оновлює лише один прапорець у конкретній комірці пам'яті, що запобігає зносу флеш-пам'яті мікроконтролера.
- **Розумна світлодіодна та звукова індикація**:
  - **Зелений**: Вода є.
  - **Плавне згасання (жовтий ➡️ помаранчевий)**: Вода закінчилася, перші секунди тиші.
  - **Блимаючий червоний + сирена**: Аварійний режим.
  - Бузер має складний нелінійний алгоритм звуків (короткий біп на старті, подвійний сигнал через 2 секунди, наростальна сирена через 5 секунд).
- **Багатозадачність (FreeRTOS)**: Робота з Wi-Fi та Telegram винесена в окремий фоновий потік, тому локальні функції безпеки (бузер, світлодіод, зчитування датчика) працюють миттєво без затримок чи зависань.

## 🔌 Схема підключення (Wiring)

Для проекту використовується наступна схема розпиновки:

| Компонент | ESP32-C3 Pin | Опис |
| :--- | :--- | :--- |
| **Датчик XKC-Y25-NPN (Yellow)** | `GPIO 4` | Зчитування рівня води (Low = є вода) |
| **Buzzer (Passive/Active)** | `GPIO 2` | Звуковий сповіщувач (через ШІМ `ledc`) |
| **Onboard WS2812 (NeoPixel)** | `GPIO 8` | Візуальна RGBW індикація статусів |

> [!NOTE]
> Схематичне зображення підключення можна переглянути у файлі [wiring_diagram.svg](wiring_diagram.svg).

## 🛠️ Налаштування та встановлення

### 1. Підготовка в Arduino IDE
Для роботи проекту вам знадобляться такі бібліотеки (встановіть через Library Manager):
- **UniversalTelegramBot** (by Gianbacchio)
- **Adafruit NeoPixel** (by Adafruit)
- **ArduinoJson** (рекомендовано версію 6.x)

### 2. Налаштування прошивки
Відкрийте файл [water_level_telegram.ino](water_level_telegram/water_level_telegram.ino) та вкажіть ваші конфігурації:
```cpp
// Wi-Fi налаштування
const char* WIFI_SSID = "ВАШ_SSID";
const char* WIFI_PASSWORD = "ВАШ_ПАРОЛЬ";

// Токен бота
const char* BOT_TOKEN = "ВАШ_ТЕЛЕГРАМ_ТОКЕН";
```

### 3. Створення та запуск Telegram-бота
1. Створіть власного бота через офіційного `@BotFather` у Telegram та отримайте унікальний **Token**.
2. Впишіть цей Token у прошивку (див. крок 2) та завантажте скетч в ESP32-C3.
3. Знайдіть вашого бота в Telegram та надішліть йому команду `/start`.
4. Скористайтеся згенерованим Reply-меню для взаємодії з пристроєм!

## 🚀 Оптимізації зв'язку
- **Без NTP-залежності**: Прошивка використовує режим `setInsecure()` для обходу перевірки дат сертифікатів. Бот починає працювати відразу після з'єднання з Wi-Fi, навіть якщо у вашій мережі заблоковано NTP-запити (UDP 123).
- **Socket Reset**: Перед кожним запитом до серверів Telegram з'єднання примусово скидається (`securedClient.stop()`). Це запобігає витоку сокетів та гарантує стабільну роботу 24/7.
